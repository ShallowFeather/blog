    FlareOn11 Writeup

    第五題

    首先要先找出他題目要求的位置，可以利用 7zip 來看時間，或利用鑑識工具 (autopsy) 就能找到 coredump

    那找到以後載入 gdb 可以發現他停在 00000000 的 address

    利用 frame 1 或 up 可以往上切

    會發現他是在使用一個 liblzma 庫 把它拉去 ida 做分析

    經過分析以後可以知道他是做 chacha20 加解密 shellcode 然後後面在 dlsym 時因為字串後面多一個空白所以找不到 function 藉而觸發 coredump

    所以要先將 shellcode 提取出來

    直接把該地址上的記憶體抓出來就好了

    shellcode 就是在做 socket 的交互

    然後會做類似 chacha20 的操作

    賽後我才發現他 expand-32 的 k 是大寫＝＝

    我是用 qiling 做模擬的

    第七題

    這題可以先用 https://github.com/ZackMount/NativeAOT-Signatures

    載入 ida 後可以比較好去做分析有些 function 會被顯示出來

    他主要邏輯為

    1. 作 WSA 網路連線
    2. 0000000140107BC0 這邊會去做自訂曲線的定義

    1. 送出一個自訂曲線上的點 (x, y)
    2. 會去 xor 0x1337 字串
    3. 驗證這個點是否在曲線上
    4. 接收一個點 (x, y)
    5. 會去 xor 0x1337 字串
    6. 驗證這個點是否在曲線上
    7. 算出共同金鑰
    8. 做 sha512(x) 拿結果做 chacha20 包含 key 跟 nonce
    9. 做加密通信

    那這題我學蠻多的主要是 c++ decompile 還有 chacha20 的部分

    基本上統整起來需要被逆向出的資訊主要是

    1. 他是在做橢圓曲線加密
    2. 自訂曲線的參數
    3. 送出的 x, y 和接收的 x, y
    4. 他的通訊流程

    啊我不會密碼學所以我找了朋友幫忙 基本上就是 **Pohlig-Hellman 但我也不熟**

    所以這邊不討論
